# КОНТЕКСТ

В проекте используется рабочий документ:

Documentation/SCHEDULE_TASK_LOOP.md

Он нужен для последующих итераций по обновлению календарных графиков занятий в DOCX-программах.

Пользователь будет работать по схеме:

1. ChatGPT готовит большой блок задачи.
2. Пользователь вставляет этот блок в CURRENT TASK.
3. Следом ChatGPT даёт короткую команду агенту.
4. Агент открывает SCHEDULE_TASK_LOOP.md, читает правила и выполняет только CURRENT TASK.

Также при открытии DOCX-документа Microsoft Word может создавать временные lock-файлы вида:

~$*.docx
~$_*.docx

Например:

Программа/~$_Ознакомительная_Бартов_2026_Зима.docx

Это не основной документ и не файл программы. Такой файл появляется, когда DOCX открыт у пользователя, и обычно удаляется автоматически после закрытия Word.

## SYSTEM INSTRUCTIONS

Агент работает как исполнитель.

Правила:

1. Читать задачу из раздела `CURRENT TASK`.
2. Выполнять только текущую задачу.

4. Не переписывать DOCX с нуля.
5. Не менять темы занятий без отдельного указания.
6. Не менять количество часов без отдельного указания.
7. Не менять количество занятий без отдельного указания.
8. Не менять второй график, если задача относится только к первому.
9. Не менять файлы вне явно указанных.
10. Сохранять форматирование DOCX.
11. Если задача аналитическая — ничего не изменять.

13. После выполнения обновить раздел `AGENT REPORT`.
14. Не удалять разделы `SYSTEM INSTRUCTIONS`, `DOCUMENT STRUCTURE NOTES`, `DATE CALCULATION RULES`, `PREVIOUS TASK`, `CURRENT TASK`, `REVIEW`, `AGENT REPORT`.

## DOCUMENT STRUCTURE NOTES

В DOCX-файле есть Приложение 2.

В Приложении 2 находятся два календарных графика занятий.

График 1:

- исходно был построен по дням недели понедельник / среда;
- содержит 32 занятия;
- даты связаны с темами занятий.

График 2:

- отдельный график для другой группы / потока;
- его нельзя автоматически менять вместе с графиком 1;
- для изменения нужны отдельные вводные: дата начала и дни недели.

## DATE CALCULATION RULES

При расчёте нового графика использовать производственный календарь РФ на соответствующий год.

Общие правила:

1. Занятия идут только по указанным дням недели.
2. Количество занятий сохраняется.
3. Если дата занятия попадает на официальный нерабочий праздничный день РФ — эту дату пропустить.
4. После пропуска продолжить график по следующему подходящему учебному дню.
5. Если рабочий день перенесён на субботу, субботу не добавлять автоматически, если группа не занимается по субботам.
6. Не додумывать новые дни занятий без указания пользователя.
7. Если есть сомнение по праздничному дню или переносу — записать это в `QUESTIONS`, а не менять график наугад.

======================================================================
                      DATE CALCULATION RULES
======================================================================

При расчёте нового графика использовать производственный календарь РФ на соответствующий год.

Общие правила:

1. Занятия идут только по указанным дням недели.
2. Количество занятий сохраняется.
3. Если дата занятия попадает на официальный нерабочий праздничный день РФ — эту дату пропустить.
4. После пропуска продолжить график по следующему подходящему учебному дню.
5. Если рабочий день перенесён на субботу, субботу не добавлять автоматически, если группа не занимается по субботам.
6. Не додумывать новые дни занятий без указания пользователя.
7. Если есть сомнение по праздничному дню или переносу — записать это в `QUESTIONS`, а не менять график наугад.

======================================================================
                         TASK FORMAT RULES
======================================================================

Для последующих итераций пользователь может давать задачу в двух сообщениях.

Сообщение 1 — большой блок задачи.

Его нужно вставлять в:

`БЛОК 1. CURRENT TASK`

Этот блок является основной задачей. Именно он определяет, что нужно сделать.

Сообщение 2 — короткая команда агенту.

Её нужно вставлять в:

`БЛОК 2. PREVIEW / SHORT COMMAND`

Этот блок не является самостоятельной задачей. Он только помогает агенту быстро понять:

- какой файл открыть;
- какие разделы прочитать;
- какие файлы можно менять;
- что нельзя трогать;
- куда вернуть отчёт.

Агент должен:

1. Открыть `Documentation/SCHEDULE_TASK_LOOP.md`.
2. Прочитать `БЛОК 1. CURRENT TASK`.
3. Прочитать `БЛОК 2. PREVIEW / SHORT COMMAND`.
4. Прочитать `SYSTEM INSTRUCTIONS`.
5. Прочитать `DOCUMENT STRUCTURE NOTES`.
6. Прочитать `DATE CALCULATION RULES`, если задача связана с датами или графиками.
7. Прочитать `TASK FORMAT RULES`.
8. Выполнить только задачу из `БЛОК 1. CURRENT TASK`.
9. Проверить результат по разделу `REVIEW`.
10. После выполнения обновить `AGENT REPORT`.
11. Вернуть отчёт в чат.

Важное правило:

Разовая задача должна находиться только в `БЛОК 1. CURRENT TASK`.

Короткая команда должна находиться только в `БЛОК 2. PREVIEW / SHORT COMMAND`.

Нельзя вставлять разовые задачи в начало файла выше `БЛОК 1. CURRENT TASK`.

Если в задаче указан коммит, но агент работает локально, коммит не означает обязательное переключение Git. Коммит нужен для ориентира. Работать нужно с текущими локальными файлами, если пользователь не дал отдельную команду переключиться на конкретный коммит.

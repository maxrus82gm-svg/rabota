## Главный принцип будущей правки

При будущей правке DOCX исправлять только перечисленные ниже места.

Не менять:

- структуру документа;
- даты;
- часы;
- номера занятий;
- темы по смыслу;
- приложения;
- общий объём программы;
- форматирование без необходимости;
- содержание, не указанное в этом плане.

Не переписывать документ литературно.

Исправлять только реальные ошибки, опечатки, пробелы, пунктуацию, согласование и единообразие терминов.

## Что пока НЕ исправлять автоматически

### UE5 / Unreal Engine 5

Не выполнять массовую замену `UE5` на `Unreal Engine 5`.

Рекомендуемое правило на будущее:

- первое упоминание: `Unreal Engine 5`;
- дальше допустимо `UE5`, если из контекста понятно, о чём речь.

---

### Assets / assets / ассеты

Не выполнять массовую замену `Assets` на `ассеты`.

Пока исправлять только явные пунктуационные ошибки:

- `Assets которые` → `Assets, которые`
- `Assets утвержденных командой` → `Assets, утверждённых командой`

Решение о едином написании `Assets`, `assets` или `ассеты` принять отдельно.
